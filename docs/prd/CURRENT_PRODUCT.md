## 2. 현재 스코프 (출시된 기능)

### 2.1 포함된 기능
- **프렌즈홈 생성**: 제목 입력으로 새 홈 생성. 첫 방문 시 `/create` 또는 메인에서 “새 프렌즈홈 만들기”.
- **프렌즈홈별 게시판**: 공지사항, 서비스 소개(About), 자유게시판, (홈이름) 소식, 임시 게시판.
- **글쓰기·글보기·댓글**: 게시판별 글 목록, 글 상세, 글쓰기, 댓글. 비회원은 닉네임을 직접 입력해 작성할 수 있다.
- **시각 표시 정책 통일**: 홈 메인의 최근 글과 동일한 상대시간 정책(`n분/시간/일/달/년 전`)을 글 상세와 댓글에도 노출한다.
- **회원가입/로그인**: 이메일·비밀번호·닉네임으로 회원가입하고 로그인할 수 있다. 로그인 사용자는 글/댓글 작성 시 계정 닉네임이 자동 적용된다.
- **비밀번호 찾기**: 이메일+닉네임이 일치하면 등록된 비밀번호를 확인할 수 있다.
- **인증 배지 표시**: 로그인(가입) 사용자가 작성한 글/댓글 닉네임 옆에 인증 아이콘을 표시한다.
- **캠핑장 미니룸 씬**: 모든 프렌즈홈 메인에 캠핑장 픽셀아트 배경을 적용한다.
- **실시간 미니룸 테마**: 현재 시각·날씨(서울 기준)에 맞춰 미니룸 배경 테마(맑음/흐림/비/눈, 주·야간)를 자동 반영한다.
- **랜덤 픽셀 캐릭터**: 캐릭터 스타일 10종을 제공하고, 홈/좌석 기준으로 랜덤 배정해 미니룸에 표시한다.
- **미니룸 캐릭터 이름표**: 미니룸 캐릭터 아래에 실제 작성자 닉네임을 표시한다. (고정 홈지기 없음)
- **미니룸 활기 지표**: 홈 메인에서 `homeId` 기준 유니크 작성자(닉네임 문자열 기준)를 집계해 방문 캐릭터 수가 늘어난다. (최대 5명 표시)
- **미니룸 휴면 표시**: 오늘(로컬 날짜 00:00 기준) 글/댓글 활동이 없는 작성자는 캐릭터 위에 `zzz`로 휴면 상태를 표시한다.
- **홈 메인 2단 게시판 영역**: 1단에 기본메뉴·커뮤니티, 2단에 최근 글 목록을 배치한다.
- **IT 모임 AI 시드 홈**: `home_ai_it_meetup` 홈(기본명: `분당IT 모임`)과 시드 게시글/댓글을 기본 제공해 초기 진입 시에도 대화 맥락을 확인할 수 있다.
- **클로드 코드 도입 논의 시드**: 분당IT 모임 자유게시판에 프러덕트 본부의 클로드 코드 도입 과정·방법·가이드·팁 주제를 기본 스레드로 제공한다.
- **IT 모임 자동 활동**: AI 시드 홈에서 30분 간격으로 글 또는 대화형 댓글(슬롯당 2개)이 자동 누적되어 참여자 간 상호작용이 유지된다.
- **AI 시드 홈 커스터마이즈**: 시드 홈 제목을 사용자가 바꾸면 사용자 지정 제목을 우선 유지한다.
- **공지사항**: 운영자 비밀번호(`OPERATOR_PASSWORD`) 입력 후에만 글쓰기 가능.
- **키보드 조작**: 1~5 메뉴, N 새 프렌즈홈, Enter 확인, ↑↓ 메뉴 이동, Tab/Esc. 하단 단축키 안내.
- **퍼머링크 공유**: 프렌즈홈 제목 옆 🔗로 현재 URL 복사.
- **동적 OG 메타(Vercel)**: `/`와 `/home/:homeId` 및 하위 링크에서 OG 제목·설명·URL을 서버에서 동적으로 주입한다. 홈 공유 시 OG 제목은 `(<홈 이름>)의 프렌즈홈에 방문해보세요`를 사용하며, 메인홈은 OG 이미지 포함, `/create`는 OG 이미지를 비운다.
- **공용 데이터 저장소**: `/api/shared-data`를 통해 홈 목록·게시글·댓글을 공용 DB(Upstash Redis)로 저장/조회하여 사용자 간 동일 데이터 노출.
- **폴백 데이터 저장소**: 공용 DB 미구성/장애 시 localStorage(홈 목록·게시글·댓글)로 자동 폴백.
- **오프라인/장애 outbox 재동기화**: 공용 API 실패 중 생성된 홈/글/댓글은 브라우저 outbox에 저장되고, API 복구 시 자동으로 원격 DB에 재반영된다.
- **동기화 상태 배너**: Header 하단에 `공용DB 연결됨 / 로컬 모드 / 확인 중` 상태를 표시해 “친구에게 안 보이는” 상황을 즉시 인지할 수 있다.
- **닉네임/인증 세션 저장**: 비회원 닉네임은 sessionStorage, 회원 목록·로그인 세션은 localStorage + 공용 DB 동기화.

### 2.2 Out of scope (현재 버전)
- 강한 권한 체계(RBAC) 및 고급 인증 정책(2FA, 비밀번호 해시 정책 등)
- 푸시·이메일 알림
- 검색·첨부파일
- GitHub Pages 환경에서 URL별 OG 메타 서버 동적 생성 (서버리스 API 미지원)

## 3. 사용자 플로우

1. **진입**: `/` → 홈이 있으면 첫 번째 홈으로, 없으면 `/create`로 이동.
2. **홈 생성**: `/create`에서 제목 입력 → 생성 후 해당 `/home/:homeId`로 이동.
3. **메인 메뉴**: 1 공지, 2 소개, 3 자유, 4 소식, 5 임시.
4. **성장 확인**: 홈 메인 캠핑장 미니룸에서 유니크 작성자 수와 캐릭터 증가 상태를 확인.
5. **글쓰기**: 게시판 진입 후 글쓰기(공지는 운영자 비밀번호 필요).
6. **공유**: 🔗로 URL 복사 → 친구에게 전달 → 같은 URL로 접속해 같은 프렌즈홈 이용.

## 6. 인수 조건 참조 (주요 시나리오)

- **AC1**: 사용자가 `/create`에서 제목을 입력해 제출하면 새 프렌즈홈이 생성되고 해당 홈 메인으로 이동한다.
- **AC2**: 1~5 키로 각 메뉴(공지·소개·자유·소식·임시)로 이동할 수 있다.
- **AC3**: 4·5번 게시판(소식·임시)은 비밀번호 없이 바로 열람·글쓰기 가능하다.
- **AC4**: 공지사항 글쓰기는 운영자 비밀번호 입력 후에만 가능하다.
- **AC5**: 공용 DB가 설정된 배포 환경에서는 홈/글/댓글이 사용자 간 공통으로 보인다. 공용 DB가 없으면 기존 localStorage/sessionStorage 동작으로 폴백한다.
- **AC6**: 같은 홈에서 유니크 작성자(닉네임 기준)가 늘면 홈 메인 캠핑장 미니룸 캐릭터 수가 증가하며, 최대 5명까지 표시된다. 오늘 활동이 없는 작성자는 `zzz` 표시를 한다.
